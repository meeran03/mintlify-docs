Conversation sessions track multi-assistant conversations as they flow through a graph. Each session maintains context, transition history, and the current state of the conversation.

## List Graph Sessions

<ParamField path="graph_id" type="integer" required>
  The unique identifier of the graph.
</ParamField>

<ParamField query="skip" type="integer" default="0">
  Number of items to skip for pagination.
</ParamField>

<ParamField query="limit" type="integer" default="100">
  Maximum number of items to return (1-1000).
</ParamField>

<ParamField query="active_only" type="boolean" default="false">
  When `true`, only returns active (ongoing) sessions.
</ParamField>

### Response

Returns an array of conversation session objects.

<ResponseField name="[]" type="array">
  Array of conversation sessions.

  <Expandable title="Session properties">
    <ResponseField name="id" type="integer">Session ID</ResponseField>
    <ResponseField name="call_sid" type="string">Unique call identifier</ResponseField>
    <ResponseField name="graph_id" type="integer">Associated graph ID</ResponseField>
    <ResponseField name="current_node_id" type="integer">Current node in the graph</ResponseField>
    <ResponseField name="context" type="object">Shared conversation context</ResponseField>
    <ResponseField name="started_at" type="string">Session start timestamp</ResponseField>
    <ResponseField name="ended_at" type="string">Session end timestamp (null if active)</ResponseField>
    <ResponseField name="is_active" type="boolean">Whether session is ongoing</ResponseField>
    <ResponseField name="current_assistant_name" type="string">Name of current assistant</ResponseField>
    <ResponseField name="graph_name" type="string">Name of the graph</ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash List Sessions
curl "https://api.burki.dev/api/v1/assistant-graphs/1/sessions?limit=10&active_only=false" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```python Python
import requests

response = requests.get(
    "https://api.burki.dev/api/v1/assistant-graphs/1/sessions",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    params={"limit": 10, "active_only": False}
)

sessions = response.json()
for session in sessions:
    status = "Active" if session['is_active'] else "Completed"
    print(f"Session {session['call_sid']}: {status} - {session['current_assistant_name']}")
```

```javascript JavaScript
const params = new URLSearchParams({
  limit: '10',
  active_only: 'false'
});

const response = await fetch(
  `https://api.burki.dev/api/v1/assistant-graphs/1/sessions?${params}`,
  {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY'
    }
  }
);

const sessions = await response.json();
sessions.forEach(session => {
  const status = session.is_active ? 'Active' : 'Completed';
  console.log(`Session ${session.call_sid}: ${status} - ${session.current_assistant_name}`);
});
```
</RequestExample>

## Get Session by Call SID

<api method="GET" endpoint="/api/v1/assistant-graphs/sessions/{call_sid}" />

Retrieve a specific conversation session by its call SID.

<ParamField path="call_sid" type="string" required>
  The unique call identifier (SID) for the session.
</ParamField>

<RequestExample>
```bash Get Session
curl https://api.burki.dev/api/v1/assistant-graphs/sessions/CA1234567890abcdef \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```python Python
import requests

response = requests.get(
    "https://api.burki.dev/api/v1/assistant-graphs/sessions/CA1234567890abcdef",
    headers={"Authorization": "Bearer YOUR_API_KEY"}
)

session = response.json()
print(f"Current assistant: {session['current_assistant_name']}")
print(f"Context: {session['context']}")
```

```javascript JavaScript
const response = await fetch(
  'https://api.burki.dev/api/v1/assistant-graphs/sessions/CA1234567890abcdef',
  {

      'Authorization': 'Bearer YOUR_API_KEY'
    }
  }
);

const session = await response.json();
console.log(`Current assistant: ${session.current_assistant_name}`);
console.log('Context:', session.context);
```
</RequestExample>

## Understanding Session Context

The `context` object in a session is a shared state that persists across assistant transitions. It's used to:

1. **Pass Information**: Share customer data between assistants
2. **Track State**: Store conversation state variables
3. **Enable Conditions**: Provide data for state variable transition conditions

### Context Example

```json
{
  "customer_name": "John Smith",
  "account_number": "12345",
  "inquiry_type": "billing",
  "issue_description": "Overcharge on last invoice",
  "sentiment": "frustrated",
  "previous_assistants": ["receptionist", "billing"],
  "escalation_requested": false,
  "call_duration_at_transition": 180
}
```

<Tip>
Assistants can read from and write to the session context during the conversation. Use descriptive key names to make the context self-documenting.
</Tip>
