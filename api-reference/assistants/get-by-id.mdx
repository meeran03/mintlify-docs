<section aria-labelledby="get-assistant-overview">
  <h2 id="get-assistant-overview">Retrieve One Assistant</h2>
  <p>
    Use this endpoint to load the current configuration for a single assistant before showing an edit screen, debugging call behavior, or syncing settings into another system.
  </p>
  <ul>
    <li>Fetch model, voice, speech, interruption, and tool settings in one response.</li>
    <li>Confirm whether an assistant is active before routing live calls to it.</li>
    <li>Use the returned ID and timestamps to compare configuration changes over time.</li>
  </ul>
</section>

<section aria-labelledby="get-assistant-api-behavior">
  <h2 id="get-assistant-api-behavior">How this endpoint behaves</h2>
  <p>
    Pass the assistant identifier you received from create or list. The response is the same assistant shape as in list responses, with the full configuration for that record.
  </p>
  <p>
    Use this call before editing an assistant in your own UI, before assigning numbers, or when debugging why a live call used certain model or voice settings.
  </p>
</section>

### Path Parameters

- `assistant_id` (string, **required**): The unique identifier for the assistant you want to retrieve. This ID is returned when you create an assistant or list all assistants.

### Response

If an assistant with the specified ID is found, the response will be a JSON object containing the full details of that assistant. This is the same object structure you receive when listing multiple assistants.
