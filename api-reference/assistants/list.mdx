<section aria-labelledby="list-assistants-overview">
  <h2 id="list-assistants-overview">List Organization Assistants</h2>
  <p>
    Use this endpoint to build assistant dashboards, selection menus, audits, and sync jobs. It returns the assistants your authenticated organization can access.
  </p>
  <ul>
    <li>Page through large assistant lists with <code>skip</code> and <code>limit</code>.</li>
    <li>Show assistant names, active status, and provider settings in your own UI.</li>
    <li>Use the returned IDs with the get, update, phone-number, and call APIs.</li>
  </ul>
</section>

<section aria-labelledby="list-assistants-api-behavior">
  <h2 id="list-assistants-api-behavior">How this endpoint behaves</h2>
  <p>
    Results include every assistant in the authenticated organization. Use <code>skip</code> and <code>limit</code> to page through large directories without loading everything at once.
  </p>
  <p>
    Each item in the array uses the same assistant object shape as the get-by-id response, so you can reuse one client model across list and detail views.
  </p>
</section>

### Query Parameters

- `skip` (integer, optional, default: 0): The number of assistants to skip before starting to collect the result set.
- `limit` (integer, optional, default: 100): The maximum number of assistants to return.

### Response

The response is a JSON array where each object represents an assistant, containing its full configuration details.
