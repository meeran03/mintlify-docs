title: Installation & Setup
---

<Callout type="info">
You can run Burki Voice AI locally, with Docker, or deploy to Railway/production. Choose the method that fits your needs!
</Callout>

---

## 1. Clone the Repository

```bash
git clone https://github.com/meeran03/burki.git
cd burki
```

---

## 2. Choose Your Setup Method

<Tabs>
  <Tab label="Local Development">

  ### Python Environment
  
  ```bash
  python3 -m venv venv
  source venv/bin/activate  # On Windows: venv\Scripts\activate
  pip install -r requirements.txt
  ```

  ### Database Setup
  
  <Accordion title="PostgreSQL (Recommended)">
  1. Install PostgreSQL (if not already installed)
  2. Create a database (e.g., `burki_db`)
  3. Update your `.env` file with the database URL
  4. Run migrations:
  
  ```bash
  alembic upgrade head
  ```
  </Accordion>

  ### Environment Variables
  
  ```bash
  cp .env.example .env
  # Edit .env with your API keys and settings
  ```

  ### Run the App
  
  ```bash
  python app/main.py
  ```
  Visit [http://localhost:8000](http://localhost:8000)

  </Tab>
  <Tab label="Docker">

  ### Build & Run
  
  ```bash
  docker build -t burki-voice-ai .
  docker run -p 8000:8000 --env-file .env burki-voice-ai
  ```
  Visit [http://localhost:8000](http://localhost:8000)

  </Tab>
  <Tab label="Railway (Cloud)">

  1. [Sign up for Railway](https://railway.app/)
  2. Connect your GitHub repo
  3. Set environment variables in Railway dashboard
  4. Deploy!
  5. Add a PostgreSQL plugin if needed

  <Callout type="success">
  Railway handles most setup for you. Just add your secrets and deploy!
  </Callout>

  </Tab>
</Tabs>

---

## 3. Environment Variables

<Accordion title="What do I need to set?">
- **Database URL** (e.g., `postgresql://user:pass@localhost:5432/burki_db`)
- **Twilio Account SID & Auth Token**
- **API keys** for LLM, TTS, and STT providers
- **Other settings** (see `.env.example` for all options)
</Accordion>

<Callout type="warning">
Never commit your `.env` file or secrets to version control!
</Callout>

---

## 4. Troubleshooting

<Accordion title="Common Issues & Solutions">
- **Database connection errors:**
  - Check your database URL and credentials
  - Make sure PostgreSQL is running
- **Missing dependencies:**
  - Run `pip install -r requirements.txt` again
- **Port already in use:**
  - Change the port in your run command or stop the conflicting process
- **API key errors:**
  - Double-check your keys and provider dashboards
</Accordion>
